    • The Process and Approch in this project

    Here I would talk about the overall design and the process of the website

    1. Picking The Basic Intergrated Template On DreamWeaver

    I have use the portfolio template four times! However, I made many modifications in terms of the structure and content.
    I used the tempate since it has a basic layout of the website: a navbar, a footer, and jumbotron. (I did use Jumbotron on almost all of the webiste, because I think this is how make the webiste look consistent. This is also what I found when mocking up the UofT website for the first assignment, in which UofT websites have a jumbotron on every inner page.)
    For the resume page, I just the basic resume to start with.
    For the content page, I believed I migrate the content part of the resume part and made it a single page.
    Resume page and the content page did not have the jumbotron. Then I felt that might make the look not consistent throught the whole website experience. Therefore, I added it and used a linear gradient background that I found from CSS-Trick website(if I do not have a good picture that fit the theme of the inner page).
    For the framework, I have make many modification espeically on button. Since most original buttons do not have shadows that interact with the operations which can potentially be an accessibility issues.

    2. Responsive Web Development

    There are two major parts. First, responsive content. Second, responsive grid. Bootstrap have a excellent responsive grid system that just work almost perfectly for everything. However, it is not the case when it comes to some specific components, which I did not expect before I just them.
    For Carousel as example, I am very enjoying using this. However, there are some problems that I need to make modifications to it. For example, it did not fit video content perfectly. I intergrated Youtube video as a carousel-item into the carousel in order to make the content more diverse. However, it did not support the scale... I have write fixed pixel heigh and width in order to scale it as expected. In other words, I need to make it responsive throug @media in the css with proper size in css. Besides, it also does not support the images with different scales well. It only work nice, if the images are all in the same proportional scale, which is not my case. I ran out of time and failed to find a way to create a image frame that could contains the images and prevent it from being scaled when displaying all the items in the same size.
    Nevertheless, I made the image responsive. Therefore, even though the scales are different, but the image and the video will be responsive for different devices and display the content properly.

    3. Content Production

    I am not the type of people who use the camera record their own life. Compare to them, I am a very techy guy who loves coding and playing video games. Therefore, I thought I might be able to take screenshots and screen recording from one of my 3D projects to demostrate the technical side. For the life side, I used the pictures from my previous working experiences, such as part-time and internship.
    I avoided using the pictures from online. Even for the icons in about me, I drew it myself with Adobe Illustration.
    My inner pages includes resume, about, contact and porfolio gallery.

    4. Design: Communication and Graphic

    I will talk about the design in the order of the website.
    The very first part of the website is the navigation bar. I think one of the best part of using a framework is the that the developers consider all of the details for us. Therefore, in order to make the best of that with the best reponsive practice, I used the one of the default navigation bar (light navigation bar) in the framework. It is not to say that I have made no modifications for it: I make the navigation bar items look like other buttons in the framework, so that the website look consistent. Other than that, I also made "jump to" function for quick navigation within each page and search function (it seems to be very hard to implement a search for the whole page).
    other very important thing about the navigation bar is that I designed it to be fixed navigation bar, which means it will stay on the top of the view no matter how the user scroll. This also causes many problems: for example, I have to add margin-top: 70px; to the html properties and all the image model properties. Otherwise, content would likely to be convered by the navigation bar.
    For the body part, I almost use container row and gird as much as possbile, so that the content would as flexible as possible for responsive design. The hardest thing to implement was Slideshow.
    One of the regret was that I do not have time to implement the scroll spy, which I think it will add many dynamic into the responsive design of the website.
    In a lot of my page, I added many links to my Gitbhub in consideration of letting the user easier to access them. I know it might not be a good practice for commerical website, because the traffics are redirected outside of the website, however, since I did that because I would like to demonstrates my project effectively.
    For the order of script, I think it can cause detrimental, unwanted, and unexpected result if we do not properly order it. For my case, I imported w3.css for the image modal function on one inner page. However, after I imported I found that many of my settings such as font and colors are changed. Therefore, I used f12 function to inspect and found that the w3.css overwrite it. At first, I was trying to commenting out the lines of code in w3.css that affect the settings, then I realized that just changed the order of importation of css change it by a lot! The same for JS! And all those code deletion in w3.css were unnecessary! Except for one line of code that changes my font and font size, which are not seems to be specified in Bootstrap framework.

    5. Web Accessibility

    I uses WAVE and DreamWeaver to check my accessibility. For Wave, it finds the errors and the colour contrast problems for me. Dreamweaver also reminds my when I do not have alt text for an image.
    One thing I notice very frustrating is that class such as "text-secondary" in the Bootstrap framework will make the text with contract errors with WAVE. Therefore, I removed some use of it and use others instead. There are also many similar cases.

    6. IA

    There are six pages in total: home , process, about, resume, portfolio, and contacts. These are at the main navigation.
    For the secondary part, since there are not much utilities, I was intented to add in-page search, which can be very useful if it works. However, I found that since I already have a jump to, which is a table of content for that page. Therefore, I felt it is a little redundant to have a in-page search.
